Data Scientist Workflow

What’s Data Discovery?

Purpose

  • relevant data insight(using pattern, statistics, visualization, etc..)
  • communicate these insights to business users
  • improve business processes

DDP(Data Discovery Platform)?

DDP가 왜 필요한가

  • “A” 데이터는 어디서 찾을 수 있나요?
  • “B” 데이터는 무엇을 의미하나요?
  • “C” 데이터는 어떻게 생성되었나요?
  • “D” 데이터를 어떻게 사용해야하죠? 주로 사용하는 사람은 누구인가요?
  • “E” 데이터 업데이트 주기를 알고싶어요

위 질문들의 경우

→ 도메인 지식이 없다면 도메인 전문가에게 데이터의 위치를 매번 물어봐야함

→ 테이블에 대한 정리가 되어있지 않은 경우 해당 테이블의 어떤 column이 있고 데이터의 type과 값이 무엇을 의미하는지 관련 전문가(도메인/분석가)에게 매번 물어봐야함

→ 문서화가 되어있지 않다면 구전으로 전해져서 데이터의 의미가 왜곡되거나 팀원 부재등의 원인으로 의미를 파악할 수 없는 데이터들이 생겨날 수 있음

→ 또한 문서화 되어있다고 하여도 매번 문서를 참조하여 의미를 파악해야하고 문서를 매번 최신화해야함

→ 데이터를 누가 만들었고, 어떤 경로에 생성되는지 등 데이터에 대한 정보를 구성원이 모두 알지 못하는 경우 생김

즉, 생산성 & 커뮤니케이션을 위해 DDP가 필요..

DDP 기능

  1. 조직 내에서 필요한 데이터가 어디 있는지 빠르게 검색
    1. 기본 방법 : 아래 예시와 같은 메타 데이터를 ElasticSearch에서 검색
      1. 테이블명
      1. 컬럼명
      1. 컬럼 설명
      1. 사용자가 입력한 설명/comment
    1. 이때, 검색 결과가 너무 많은 경우에는 우선순위를 매기는게 필요
      1. Lyft, Spotify : 테이블 사용량을 랭킹정보로 넣음 → 쿼리 로그를 파싱
      1. Nemo : 검색어를 spaCy로 자연어 파싱, 소셜그래프 검색에 사용하는 Unicorn으로 kNN기반 스코어링등을 통해 랭킹 결정
      1. 또다른 방법은 추천, 조직과 팀내에서 가장 많이 접근하고 사용하는 데이터
  1. 데이터가 어떤 건지 이해
    1. 테이블을 이해하는 기본 정보 : 데이터 스키마
      1. 컬럼 명
      1. 데이터 타입
      1. 설명(Description)
    1. 사용자가 읽기 권한이 있다면 데이터의 미리보기(100줄 정도) 제공 가능
    1. 미리 계산된 컬럼 단위의 통계도 제공
      1. column별 row 수
      1. Null인 row 수
      1. Min/Max/Mean/Median/Std
      1. 고유한(Unique) Row 수
      1. 전체 기간 범위 : 날짜 데이터
    1. 데이터 계보를 제공 → 앞뒤 의존성 관계도 확인
      1. ETL 작업(Airflow로 스케줄링)이 경우 스케줄/지연 여부등을 확인
  1. 데이터 사용(활용) 방법을 배우게 해줌 & 분석 환경의 모든 테이블의 최신 명세/통계
    1. 데이터 소유자는 권한 부여 가능, 자주 사용하는 사람들이 데이터에 대한 소개 or 특이점 알려줄 수 있음 → but 병목현상 생길 수 있음 : 메타데이터를 연결해주는것이 더 Scalable한 방식
    1. Amundsen & DataHub : 테이블에 사람을 엔티티로 연결하는게 가능
    1. 사용자가 가장 관련성이 높은 컬럼을 찾을 수 있도록 각 열의 사용통계를 제공
    1. 해당 테이블과 조인할 테이블이 어떤건지 알기위해, 주로 조인되는 테이블 목록 & 조인하는 열 제공하면 편리 → 이런 데이터들은 쿼리 로그 파싱을 필요로함
    1. 데이터 사용에 대한 더 자세한 정보를 제공하기 위해서는 해당 테이블에 대한 최근 쿼리 제공 가능 & 실제로 어떤 데이터를 필터링 하는지 알 수 있음
    1. 데이터 계보(Lineage)도 유용 : 현재 테이블을 사용하는 다운스트림 테이블에 대해 알 수 있고, 생성하는 쿼리도 보여줌 → 테이블이 다른 유스케이스에 어떻게 이용되는지 확인 가능 / 다운스트림 테이블이 검색 목적과 일치할 경우 계산/저장 비용을 줄일 수 있게됨(Twitter의 Data Access Layer, Uber의 Databook, Netflix의 Metacat 등 지원)
    1. 데이터를 실제로 사용하기 전에 얼마나 자주 업데이트 되는지 명세 → 어떤 시간단위(일/Hour 등)의 파티션 명시
    1. DDP는 Airflow 같은 플랫폼과 연계해서 스케줄링된 ETL작업 조사 & 얼마나 걸리는지 볼 수 있음

DDP 비교

  1. DataHub(LinkedIn) : https://datahubproject.io/
    • UI가 개발외 직군에게도 쉽게 활용
    • 검색, 테이블스키마, 오너십, 계보 기능 지원
    • Dataset, User, Group 세개의 엔티티 지원. Schemas/Jobs/Metrics/Dashboards 등의 엔티티 추가 예정
    • Hive, Kafka, RDB 메타데이터 지원 ( 내부에선 더 많이 지원하며, 더 공개될 수도 있음 )
    • Expedia, TypeForm 등이 채택했으며 MS,Morgan Stanley, Orange Telecom, ThoughtWorks 등도 POC 중
    • 각 테이블의 컬럼들의 통계 정보를 확인할 수 있으며 쿼리를 하지 않아도 테이블 row 수, 샘플 데이터 등을 조회 하는 것이 가능
    • Event-based 방식
  1. Amundsen(Lyft) : https://github.com/amundsen-io/amundsen
    • 검색, 추천, 미리보기/컬럼통계/소유자/주사용자 들이 잘 표현된 테이블 상세 페이지 지원. 계보기능은 없지만 추가예정
    • Data Quality System 과의 연동도 제공 예정 (아마도 Great Expectations - https://greatexpectations.io/)
    • 훌륭한 커뮤니티가 있음 : BigQuery/Redshift/Apache Atlas 등의 연동등을 개발해서 기여함
    • 15종 이상의 데이터 소스( Redshift, Cassandra, Hive, Snowflake 및 각종 RDB), Tableau,Redash, Mode Analytics 대시보드, Airflow 등과 연동을 지원
    • 문서화도 잘 되어있고, Docker 로 로컬에서 테스트 가능(python으로 개발)
    • Monolith 방식
  1. Metacat(Netflix) : https://github.com/Netflix/metacat
    • 검색, 스키마 보기, 비용과 공간을 분석하는 메트릭 포함.
    • 테이블/파티션 변경에 대한 노티 알림 기능이 있음. 비용등의 문제로 지워질때 알림을 받는게 가능.
    • Hive, Teradata, Redshift, S3, Cassandra, RDS 연동 지원
    • 스키마/메타데이타의 버저닝/검증 기능 작업중이라고 밝힌바 있음.
    • 오픈소스 이긴 하지만, 문서가 하나도 없어서 도입한 회사는 없음.
  1. Marquez(WeWork) : https://github.com/MarquezProject/marquez
    • 데이터 품질과 계보에 집중
    • 데이터 거버넌스, Great Expectations 를 통한 데이터 품질, 데이터셋과 잡에 대한 카탈로그 지원
    • WebUI 및 Airflow 컴포넌트 및 Java/Python 클라이언트 제공
    • Docker로 로컬에서 테스트 가능하지만 문서는 많지 않음
  1. Apache Atlas(Hortonworks) : https://atlas.apache.org/
    • 주 목표는 데이터 거버넌스로 조직이 보안/컴플라이언스 요구사항을 준수하도록 도와주는 것.
    • 자원에 대한 태깅, 다운스트림 데이터셋에 대한 태그 전파, 메타 데이터 접근에 대한 보안등 풍부한 기능을 가지고 있음
    • 메타데이터 변경 알림 기능도 가능
    • free-text 검색, 스키마 상세 보기, 데이터 계보 지원
    • 고급 검색 기능으로 SQL 과 비슷한 구문으로 검색하는 것도 지원
    • HBase, Hive, Kaflka 등의 메타데이터 소스와 연동 지원
    • 메타데이터를 REST API로 생성/수정도 가능
    • 문서화도 잘 되어있음
  1. Nemo(Facebook) : https://engineering.fb.com/2020/10/09/data-infrastructure/nemo/
    • 검색어를 spaCy로 자연어 파싱, 소셜그래프 검색에 사용하는 Unicorn으로 kNN기반 스코어링등을 통해 랭킹 결정
    • 오픈소스 공개 X
